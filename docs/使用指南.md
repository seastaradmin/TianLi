## 📡 API 接口

### 查看 API 文档

```
http://localhost:8000/docs
```

**ReDoc:**
```
http://localhost:8000/redoc
```

### 主要接口

#### 1. 交付物接口

```bash
# 获取所有交付物
curl http://localhost:8000/api/deliverables

# 筛选 PPT 文件
curl http://localhost:8000/api/deliverables?filter=pptx

# 下载交付物
curl -O http://localhost:8000/api/deliverables/1/download
```

#### 2. 日志接口

```bash
# 获取日志
curl http://localhost:8000/api/logs

# 筛选错误日志
curl http://localhost:8000/api/logs?level=error

# 按任务 ID 筛选
curl http://localhost:8000/api/logs?task_id=ppt-task-001

# 实时日志流 (SSE)
curl -N http://localhost:8000/api/logs/stream
```

#### 3. 指标接口

```bash
# 获取系统指标
curl http://localhost:8000/api/metrics

# 获取 Hero 性能
curl http://localhost:8000/api/metrics/hero-performance
```

#### 4. 数据库接口

```bash
# 数据库状态
curl http://localhost:8000/api/database/status

# 反馈统计
curl http://localhost:8000/api/database/feedback-stats
```

#### 5. 健康检查

```bash
# 健康检查
curl http://localhost:8000/health

# 根路径
curl http://localhost:8000/
```

## 💾 数据库

### 数据库信息

- **数据库名:** tianli_feedback
- **主机:** localhost:3306
- **用户:** root
- **密码:** (空)

### 初始化数据库

```bash
cd ~/Desktop/TianLi
python3 database/setup_database.py
```

### 查看表结构

```sql
USE tianli_feedback;

SHOW TABLES;
-- dispatch_decisions
-- task_results
-- user_feedback
-- hero_performance
-- matching_weights
-- synonym_mappings
-- ab_test_results

DESCRIBE dispatch_decisions;
DESCRIBE task_results;
DESCRIBE user_feedback;
```

### 查询示例

```sql
-- 查看最近的 dispatch 决策
SELECT * FROM dispatch_decisions 
ORDER BY created_at DESC 
LIMIT 10;

-- 查看任务执行结果
SELECT task_id, status, execution_time_ms, l2_score 
FROM task_results 
ORDER BY created_at DESC 
LIMIT 10;

-- 查看 Hero 性能
SELECT hero_id, total_tasks, successful_tasks, 
       ROUND(successful_tasks * 100.0 / total_tasks, 2) as success_rate
FROM hero_performance
ORDER BY total_tasks DESC;

-- 查看用户反馈
SELECT rating, COUNT(*) as count
FROM user_feedback
GROUP BY rating;
```

## 🔧 配置

### 数据库配置

**文件:** `tianli_harness/core/db_connector.py`

```python
config = {
    'host': 'localhost',
    'port': 3306,
    'user': 'root',
    'password': '',  # 空密码
    'database': 'tianli_feedback',
    'charset': 'utf8mb4'
}
```

### 后端配置

**文件:** `backend/main.py`

```python
# CORS 配置（允许前端访问）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",
        "http://localhost:3000"
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 🐛 故障排查

### 后端启动失败

**错误:** `ModuleNotFoundError: No module named 'fastapi'`

**解决:**
```bash
pip3 install fastapi uvicorn pymysql --break-system-packages
```

### 数据库连接失败

**错误:** `Can't connect to MySQL server`

**解决:**
1. 检查 MySQL 服务是否运行
   ```bash
   sudo systemctl status mysql
   # 或
   brew services list | grep mysql
   ```

2. 启动 MySQL 服务
   ```bash
   sudo systemctl start mysql
   # 或
   brew services start mysql
   ```

3. 重新初始化数据库
   ```bash
   python3 database/setup_database.py
   ```

### 前端无法连接后端

**错误:** `Failed to fetch`

**解决:**
1. 检查后端是否运行
   ```bash
   curl http://localhost:8000/health
   ```

2. 检查 CORS 配置
   - 确保后端允许前端端口访问
   - 检查 `backend/main.py` 中的 CORS 配置

3. 检查 API 地址
   - 确保前端调用的是 `http://localhost:8000`
   - 不是 `https://` 或其他端口

## 📝 常见问题

### Q: 如何添加新的交付物？

A: 交付物由天理系统自动生成。当执行 PPT 生成任务时，会自动创建交付物记录。

```python
# 执行 PPT 生成任务
from tianli_harness import HarnessEngine

engine = HarnessEngine(config, client, executor)
result = await engine.run("task-001", "生成产品宣讲 PPT")

# 交付物会自动记录到数据库
```

### Q: 如何查看实时日志？

A: 有两种方式：

1. **前端页面:** 访问 http://localhost:5173/live-logs
2. **API 接口:** 
   ```bash
   curl -N http://localhost:8000/api/logs/stream
   ```

### Q: 如何导出日志？

A: 在实时日志页面点击"导出"按钮，或：

```bash
curl http://localhost:8000/api/logs > logs.json
```

### Q: 数据库数据如何清理？

A: 可以手动清理或重置数据库：

```sql
-- 清理所有数据
TRUNCATE TABLE dispatch_decisions;
TRUNCATE TABLE task_results;
TRUNCATE TABLE user_feedback;
TRUNCATE TABLE hero_performance;

-- 或重置整个数据库
DROP DATABASE tianli_feedback;
CREATE DATABASE tianli_feedback;
python3 database/setup_database.py
```
